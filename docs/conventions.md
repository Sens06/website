# Правила и стек

Этот файл — единый источник правды по тому, **как** собираются сайты на этой фабрике. Агент `/build-site` обязан следовать ему.

## Стек

- **HTML5** — семантическая разметка, `lang` под язык клиента.
- **CSS3** — кастомные свойства (`--color-*`) для темизации, Flexbox/Grid для раскладки. Один файл `css/styles.css`, без препроцессоров.
- **Vanilla JS (ES6+)** — без фреймворков и зависимостей. Один файл `js/main.js`.
- **Без сборщиков** — никаких npm, webpack, Vite. Файлы открываются и публикуются как есть.
- **Хостинг** — GitHub Pages (статика). См. `docs/deploy.md`.

Причина минимализма: сайты публикуются на GitHub Pages без бэкенда и должны открываться без сборки.

## Структура собранного сайта

```
index.html          # одностраничный лендинг
css/styles.css       # все стили
js/main.js           # вся логика
assets/              # лёгкие картинки/иконки (тяжёлое медиа — НЕ в git, см. .gitignore)
```

## Правила вёрстки

- Мобильная адаптивность обязательна: `meta viewport` + проверка раскладки на узком экране.
- Темизация только через CSS-переменные в `:root` — цвета, отступы, max-width менять в одном месте.
- Семантические теги: `header`, `nav`, `main`, `section`, `footer`. Каждая секция — с `id` для якорной навигации.
- Доступность: контраст текста, `alt` у картинок, фокус-стили у интерактивных элементов.

## Заявки (лиды) — важно

GitHub Pages **не имеет бэкенда**, обычная `<form>` сама письмо не отправит. Способы сбора заявок (выбирается в брифе):

- **Formspree / Getform** — `<form action="https://formspree.io/f/XXXX" method="POST">`. Проще всего, бесплатный тариф.
- **Telegram-бот** — отправка через Bot API из `main.js` (токен виден в коде — использовать бот только для приёма заявок, без прав).
- **mailto / ссылка на мессенджер** — как запасной вариант (WhatsApp/Telegram-ссылка, тел.).

Если способ в брифе не указан — спросить клиента, не ставить «заглушку-форму», которая молча теряет заявки.

## Чек-лист перед сдачей

- [ ] Все секции из брифа на месте, плейсхолдеров не осталось.
- [ ] Форма заявки реально отправляет (проверено), есть сообщение об успехе.
- [ ] Адаптив: корректно на 360px и на десктопе.
- [ ] Якорная навигация и плавная прокрутка работают.
- [ ] `title`, `meta description`, контакты заполнены.
- [ ] Нет ошибок в консоли браузера.
