# clatz-brief-bot — Telegram-бот брифа для клиентов

Бот, которому менеджер после созвона отправляет клиента для заполнения брифа.
Работает как Cloudflare Worker (webhook), состояние хранится в KV — клиент
может закрыть чат и вернуться позже, все ответы сохраняются.

## Что умеет

- **Согласие** — перед стартом фиксирует согласие на обработку ПДн
  и условия работы (кнопка «Принимаю», сохраняется дата и время).
- **Пошаговый бриф** (22 шага): контакты (имя, телефон с кнопкой
  «поделиться контактом», e-mail, компания) → ниша → все ссылки (сайт,
  соцсети) → опыт в нише и опыт продвижения → загрузка логотипов
  (файлы/фото/ссылка на диск) → конкуренты → 3 вопроса по ведению
  контента → 8 рекламных каналов с бюджетами за последние 3 месяца →
  вопрос о публикации записи исследования в соцсетях агентства.
- **Возврат к шагам** — команда `/edit` или кнопка «Изменить ответы»:
  клиент выбирает любой заполненный шаг, меняет ответ и возвращается
  туда, где остановился. При повторном `/start` бот предлагает
  продолжить с места остановки.
- **Напоминание** — если бриф не закончен и клиент молчит 12 часов,
  cron-триггер шлёт одно напоминание (сбрасывается при любой активности).
- **Доставка** — готовый бриф + все загруженные логотипы прилетают
  в чат менеджера (`ADMIN_CHAT_ID`), перед отправкой клиент видит сводку
  и подтверждает её.

## Развёртывание

### 1. Создай бота

В Telegram у [@BotFather](https://t.me/BotFather): `/newbot` → имя, например
`К.Л.А.Ц | Бриф` и username вида `clatz_brief_bot`. Сохрани **токен**.

Там же желательно настроить команды: `/setcommands` →

```
start - Начать или продолжить бриф
edit - Изменить заполненные ответы
help - Что умеет бот
```

### 2. Узнай ADMIN_CHAT_ID

Куда слать готовые брифы. Проще всего: напиши новому боту `/start` со своего
аккаунта (или добавь его в рабочую группу), затем открой
`https://api.telegram.org/bot<ТОКЕН>/getUpdates` — в ответе будет
`"chat":{"id": ...}`. Это и есть id (у групп он отрицательный).

### 3. Создай KV и задеплой

```bash
cd brief-bot
npm install
npx wrangler kv namespace create BRIEF_KV
# → выведет id — впиши его в wrangler.jsonc вместо ЗАМЕНИ_НА_ID_ИЗ_WRANGLER

npx wrangler secret put BOT_TOKEN       # токен от BotFather
npx wrangler secret put ADMIN_CHAT_ID   # id чата менеджера
npx wrangler secret put WEBHOOK_SECRET  # любая случайная строка, придумай сам

npx wrangler deploy
# → выведет URL вида https://clatz-brief-bot.<аккаунт>.workers.dev
```

### 4. Привяжи webhook

Подставь свой токен, URL воркера и тот же WEBHOOK_SECRET:

```bash
curl "https://api.telegram.org/bot<ТОКЕН>/setWebhook" \
  -d "url=https://clatz-brief-bot.<аккаунт>.workers.dev/webhook" \
  -d "secret_token=<WEBHOOK_SECRET>" \
  -d "drop_pending_updates=true"
```

Проверка: `https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo` — поле `url`
должно совпадать, `last_error_message` — отсутствовать.

### 5. Проверь сценарий

1. Напиши боту `/start`, пройди пару шагов, закрой чат, снова `/start` —
   бот должен предложить продолжить с того же места.
2. `/edit` — изменение любого пройденного шага и возврат.
3. Дозаполни до конца — бриф и логотипы должны прийти в `ADMIN_CHAT_ID`.
4. Напоминание можно не ждать 12 часов: cron видно в дашборде Cloudflare
   (Workers → clatz-brief-bot → Triggers), логи — `npx wrangler tail`.

## Как выглядит поток для клиента

```
/start → согласие ПДн → имя → телефон → e-mail → компания → ниша →
ссылки → опыт в нише → опыт продвижения → логотипы → конкуренты →
контент (кто ведёт / форматы / пожелания) →
8 каналов: таргет ВК, Директ, TG Ads, блогеры, SEO, Авито,
офлайн, другие (бюджет/мес за последние 3 месяца или «Не использовали») →
публикация исследования (да / обсудим / нет) →
сводка → «Отправить бриф» → 🎉
```

## Файлы

- `worker.js` — весь код бота (webhook + cron), без зависимостей
- `wrangler.jsonc` — конфиг воркера: KV-биндинг `BRIEF_KV`, cron `*/30 * * * *`
- Ссылка на политику конфиденциальности в согласии: `https://clatz.ru/privacy.html`
  (константа `PRIVACY_URL` в начале `worker.js`)
