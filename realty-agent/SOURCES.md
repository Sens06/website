# Реестр источников коммерческой недвижимости — Санкт-Петербург

> Этап 1 агента поиска коммерческой недвижимости СПб. Исчерпывающий каталог мест,
> где публикуется коммерческая недвижимость (аренда/покупка/ГАБ/земля), с пометкой
> для каждого источника: открытый он или закрытый и **как реально считывается**.

## Легенда

**Режим доступа (как считывать):**

| Код | Что значит | Инструмент считывания |
|-----|------------|-----------------------|
| **A** | Официальный API / открытый машиночитаемый фид (XML/JSON) | обычный `fetch`, ключ API если требуется |
| **B** | Открытый сайт, но с антиботом/JS-рендером — нужен рендер-парсинг | ScrapingBee (наш «webee»-токен) или headless-браузер |
| **C** | Закрытый канал/база — нет открытого HTTP-доступа | ручной экспорт, Telegram API, партнёрский доступ |

**Антибот/сложность:** 🟢 низкая · 🟡 средняя · 🔴 высокая (жёсткая защита, капчи, поведенческий анализ).

> ⚠️ **Правовая пометка.** Парсинг чужих классифайдов (Авито/ЦИАН/Яндекс/Домклик)
> нарушает их пользовательские соглашения и юридически «серый». Приоритет отдаётся
> официальным API/фидам и **прямым базам брокеров/УК** — они публикуют свои объекты
> сами, для привлечения клиентов, поэтому их считывание наименее рискованно.
> Режим B по классифайдам используем осознанно и с ограничением частоты.

## 5. Закрытые каналы (режим C)

Значимая часть предложения (особенно эксклюзивы и «до публикации») ходит в закрытых
каналах. Автоматическое считывание — через Telegram API либо ручной экспорт.

| Канал/база | Тип | Как получить доступ | Как считывать |
|------------|-----|---------------------|---------------|
| **Telegram-каналы/чаты брокеров СПб по коммерции** | публичные и закрытые ТГ-каналы | подписка; для закрытых — приглашение | Telegram Bot API / MTProto (userbot) на публичные; закрытые — только ручной экспорт |
| **Winner** | профессиональная риелторская база СПб | платная подписка агентства | партнёрский доступ/экспорт |
| **База БН (профкабинет)** | закрытая часть bn.ru для риелторов | подписка риелтора | партнёрский доступ |
| **Риелторские CRM/мультилистинги** | межагентские базы | членство в объединении/подписка | партнёрский API/экспорт |
| **Профильные закрытые чаты (WhatsApp/TG)** | межброкерский обмен | личные связи | ручной сбор |

> Закрытые каналы дают эксклюзивы, но **не масштабируются автоматически** без
> партнёрских договорённостей. На Этапе 2 закладываем ручной/полуавтоматический
> ввод таких объектов в общую схему (см. `FILTER_SPEC.md`).

## Сводка по режимам (для Этапа 2)

- **Режим A (официально)** — гео/справочные API (Яндекс.Карты, ФИАС, 2GIS), при
  наличии — партнёрский API Авито. Приоритет по чистоте и легальности.
- **Режим B (ScrapingBee)** — основная рабочая лошадка: прямые базы брокеров/УК
  (низкий риск) в первую очередь, классифайды (высокий антибот, ограничение частоты)
  во вторую. Токен «webee» = ScrapingBee API-ключ, `SCRAPINGBEE_KEY`.
- **Режим C (закрытые)** — Telegram API + ручной ввод; масштабируется через
  партнёрские доступы.

Приоритет наполнения адаптеров: **брокеры/УК (§3–4) → спец-порталы (§2) →
классифайды (§1) → закрытые каналы (§5)**.
