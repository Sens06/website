# Спецификация фильтрации и схемы данных

> Этап 1. Как из разнородных источников (см. `SOURCES.md`) привести объекты к
> единой схеме и отфильтровать под конкретную цель по параметрам: **гео, площадь/
> метраж, цель/назначение, этажность, тип операции, бюджет**.

## 1. Нормализованная схема объекта (`Listing`)

Каждый объект из любого источника приводится к единому виду. Поля, которые не
удалось извлечь, остаются `null` — фильтрация это учитывает (см. §4).

```jsonc
{
  "id": "cian:315928471",           // <source>:<native_id>, уникальный
  "source": "cian",                  // ключ источника из SOURCES.md
  "url": "https://spb.cian.ru/...",  // ссылка на объявление
  "scrapedAt": "2026-07-15T10:00:00Z",

  "operation": "rent",               // rent | sale | gab (готовый арендный бизнес)
  "purpose": ["retail", "catering"], // назначение, массив (см. §3, справочник)
  "propertyClass": "B",              // A | A+ | B | B+ | C | null (для офисов/БЦ)

  "area": 95.0,                      // площадь помещения, м² (число)
  "floor": 1,                        // этаж объекта (число; -1 = цоколь/подвал)
  "floorsTotal": 9,                  // этажность здания (число | null)

  "geo": {
    "district": "Центральный",       // район СПб
    "metro": ["Маяковская"],         // ближайшее метро (массив)
    "metroDistanceMin": 5,           // пешком, мин (число | null)
    "address": "Невский пр., 100",   // сырой адрес
    "lat": 59.9311, "lon": 30.3609,  // координаты (после гео-нормализации, режим A)
    "line": "first"                  // first | inside | null (первая линия / внутри)
  },

  "price": {
    "value": 190000,                 // ставка/цена, ₽
    "period": "month",               // month (аренда) | total (продажа/ГАБ)
    "perSqm": 2000,                  // ₽/м² (в мес для аренды), вычисляется
    "currency": "RUB",
    "nds": "included",               // included | excluded | usn | null
    "opexIncluded": false            // включены ли эксплуатационные расходы
  },

  "features": {                      // характеристики, критичные для matching (§5)
    "ceilingHeight": 3.2,            // высота потолков, м | null
    "powerKw": 30,                   // выделенная электромощность, кВт | null
    "ventilation": true,             // приточно-вытяжная вентиляция | null
    "wetPoints": true,               // мокрые точки/канализация под общепит | null
    "gateType": null,                // dock | ramp | ground | null (склад)
    "separateEntrance": true,        // отдельный вход | null
    "showcaseWindows": true,         // витринные окна | null
    "parking": true,                 // парковка | null
    "condition": "fit-out"           // shell | fit-out | turnkey | null (отделка)
  },

  "traffic": {                       // пешеходный/авто трафик (для ритейла/общепита)
    "footfall": "high",              // high | medium | low | null
    "source": "estimate"             // estimate | 2gis | measured
  },

  "contact": { "name": null, "phone": null, "isAgency": true },
  "raw": {}                          // исходные поля источника, на всякий случай
}
```

**Правила нормализации:**

- `price.perSqm` = `price.value / area` (для аренды — за месяц). Если источник
  даёт ставку за м²/год — конвертировать в месяц.
- `purpose` нормализуется из текста объявления по словарю синонимов (§3).
- `geo.district` / `geo.metro` — из адреса через геокодер (режим A, Яндекс/ФИАС).
- `features.*` парсятся из описания эвристиками; отсутствие ≠ «нет», а `null`
  («неизвестно») — это влияет на строгость фильтра (§4).

## 2. Параметры запроса (`SearchQuery`)

То, что задаёт пользователь. Все поля необязательны, кроме отсутствующих ограничений
(незаданный фильтр не отсекает).

```jsonc
{
  "operation": "rent",               // rent | sale | gab
  "purpose": "catering",             // целевое назначение (одно, см. §3)

  "geo": {
    "districts": ["Центральный", "Адмиралтейский"],  // ИЛИ
    "metro": ["Маяковская"],                          // ИЛИ
    "center": { "lat": 59.93, "lon": 30.36 },         // радиус-поиск
    "radiusM": 1500
  },

  "areaMin": 80, "areaMax": 130,     // м²
  "floorMin": 1, "floorMax": 1,      // этаж объекта (для стрит-ритейла = 1)
  "floorsTotalMin": null, "floorsTotalMax": null,  // этажность здания

  "priceMax": 250000,                // потолок ставки/цены, ₽
  "perSqmMax": null,                 // потолок ₽/м²

  "requireFeatures": ["wetPoints", "ventilation"],  // жёсткие требования
  "propertyClass": null,             // для офисов

  "strictness": "balanced"           // strict | balanced | loose (см. §4)
}
```

## 3. Справочник назначений (`purpose`)

| Код | Назначение | Синонимы в объявлениях |
|-----|-----------|------------------------|
| `office` | офис | офис, кабинет, БЦ, бизнес-центр, коворкинг |
| `retail` | торговое / стрит-ритейл | торговое, магазин, street-retail, ТЦ, торговая площадь |
| `catering` | общепит | кафе, ресторан, бар, столовая, общепит, ПСН под еду |
| `warehouse` | склад | склад, складское, логистика, хранение |
| `production` | производство | производственное, цех, промышленное |
| `psn` | свободное назначение (ПСН) | ПСН, свободного назначения, универсальное |
| `services` | услуги/бьюти/мед | салон, клиника, барбершоп, фитнес |
| `hotel` | гостиница/апарт | отель, хостел, апарт-отель |
| `land` | земля под коммерцию | участок, земля, ИЖС-коммерция, промземля |
| `gab` | готовый арендный бизнес | ГАБ, арендный бизнес, с арендатором |

Назначение из объявления может быть кратным (напр. `["retail","catering"]` —
«помещение под магазин или кафе»).

## 4. Строгость фильтра (`strictness`) — как трактовать `null`

Ключевая проблема: у объявлений часто нет части характеристик. Режим строгости
определяет, отсекать ли объект с неизвестным (`null`) обязательным признаком.

| Режим | Поведение при `null` в обязательном признаке | Когда использовать |
|-------|----------------------------------------------|--------------------|
| `strict` | отсекать (нужны подтверждённые признаки) | когда критично (общепит с мокрыми точками) |
| `balanced` (по умолч.) | оставлять, но помечать `needsVerification: true` и понижать ранг | обычный поиск |
| `loose` | оставлять без штрафа | широкий охват, ручная фильтрация после |

Диапазоны (`areaMin/Max`, `floor`, `price`) применяются **жёстко** всегда — если
значение известно и вне диапазона, объект отсекается независимо от строгости.

## 5. Правила соответствия «объект ↔ цель» (matching)

Помимо диапазонов, под каждую цель есть **обязательные** и **желательные** признаки.
Обязательные при `strict` отсекают, при `balanced` понижают ранг. Желательные
только повышают ранг (скоринг).

| Цель (`purpose`) | Обязательные признаки | Желательные (бонус к рангу) | Типичные диапазоны |
|------------------|-----------------------|------------------------------|--------------------|
| **catering** (общепит) | `wetPoints`, `ventilation`, `powerKw ≥ 15` | `floor=1`, `separateEntrance`, `line=first`, `footfall=high`, вытяжка на кровлю | 50–250 м² |
| **retail** (стрит-ритейл) | `floor=1`, `separateEntrance` | `line=first`, `showcaseWindows`, `footfall=high`, метро ≤ 7 мин | 30–300 м² |
| **office** (офис) | — (гибко) | `propertyClass` A/B, парковка, метро ≤ 10 мин, `condition=fit-out` | 20–1000 м² |
| **warehouse** (склад) | `floor=1` (или наземный), `gateType` (dock/ramp) | `ceilingHeight ≥ 6`, `powerKw` высокое, пандус, ж/д ветка | 100–5000+ м² |
| **production** (производство) | наземный, `powerKw` высокое | `ceilingHeight ≥ 5`, кран-балка, отдельный въезд, промзона | 100–5000+ м² |
| **services** (салон/мед) | `wetPoints` (для бьюти/мед) | `floor=1`, отдельный вход, вентиляция | 30–200 м² |
| **hotel** | много `wetPoints`, `powerKw` | отдельный вход, несколько этажей, парковка | 200+ м² |
| **land** | — | назначение участка, коммуникации, подъезд | по проекту |
| **gab** (арендный бизнес) | действующий арендатор, договор | окупаемость (лет), надёжный арендатор, ставка капитализации | по доходности |

**Скоринг (Этап 2):** итоговый ранг = соответствие диапазонам (жёстко) →
обязательные признаки (штраф/отсев по строгости) → сумма бонусов за желательные →
свежесть объявления → цена за м² относительно медианы по району/назначению.

## 6. Псевдокод фильтра (для реализации на Этапе 2)

```js
function filterAndRank(listings, query) {
  return listings
    // 1. Жёсткие диапазоны (известное значение вне диапазона => отсев)
    .filter(l => inRange(l.area, query.areaMin, query.areaMax))
    .filter(l => matchOperation(l.operation, query.operation))
    .filter(l => matchPurpose(l.purpose, query.purpose))          // §3
    .filter(l => matchGeo(l.geo, query.geo))                       // район/метро/радиус
    .filter(l => inRange(l.floor, query.floorMin, query.floorMax))
    .filter(l => l.price?.value == null || l.price.value <= (query.priceMax ?? Infinity))
    // 2. Обязательные признаки цели с учётом строгости (§4, §5)
    .map(l => applyRequiredFeatures(l, query))                     // отсев|штраф|verify-флаг
    .filter(Boolean)
    // 3. Скоринг по желательным признакам и рынку (§5)
    .map(l => ({ ...l, score: scoreListing(l, query) }))
    .sort((a, b) => b.score - a.score);
}
```

`matchGeo` использует режим A (геокодер) для радиус-поиска; при отсутствии координат
падает на сравнение по `district`/`metro`.

## 7. Что нужно от источника, чтобы matching работал

Минимально извлекаемые поля для полезной фильтрации: `operation`, `purpose`, `area`,
`floor`, `geo.district`/`geo.metro`, `price.value`. Признаки из `features.*` —
«бонусные»: чем больше извлечём (парсингом описания или из карточки брокера), тем
точнее matching. Прямые базы брокеров/УК (§3–4 в `SOURCES.md`) дают `features`
надёжнее классифайдов — ещё одна причина их приоритета.
